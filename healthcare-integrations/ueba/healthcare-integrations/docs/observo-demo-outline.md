# Observo Data Pipelines — Demo Outline for German Healthcare M&A

**Audience:** CISO + SOC manager + IT-integration lead at a DE hospital group, MVZ rollup, or HIS/PIS vendor going through M&A.
**Total runtime:** 60 minutes (six 8–10 minute demos + 5 min wrap).
**Pre-requisites:** the artefacts in `healthcare-integrations.zip` already deployed:
- SDL parsers `Avelios-Medical-OCSF` and `Omniconnect-OCSF`
- Dashboard `/dashboards/bsi-nis2-healthcare-overview`
- An Observo tenant with an S3 source pointing at a MinIO bucket loaded with the raw `*.ndjson.gz` tree
- A second SIEM destination (Microsoft Sentinel free tier or Splunk Cloud trial) to demonstrate dual-fan-out

The demo tells **one M&A story** — *"Hospital Group A is acquiring Hospital Group B"* — and walks the customer through the timeline: Day -30 (diligence) → Day 1 → Day 90 → Year 1.

## Demo 1 (8 min) — Pre-deal Telemetry Diligence
*Maps to GTM motion #1, technical use case 3.1.*

**Story.** "Before you sign, you want to score the target's cyber posture without ingesting raw PHI."

**Steps.**
1. Open Observo. Show **S3 source** consuming `raw-logs/avelios-medical/2026/05/08/*.ndjson.gz` (the seller's hospital).
2. Show a **redact processor** chain:
   - drop `patient.kvnr`, `patient.name`, `patient.diagnosis`
   - SHA-256 hash `user.email`, `card.iccsn`, `agent.serial`
   - keep `event_category`, `event_type`, `severity`, `outcome`, MITRE tags
3. **Destination:** "deal-room S3" (a separate bucket). Open one redacted file in MinIO browser — show that PHI is gone but security signal is intact.
4. Show a 1-page risk-score PDF (pre-built) generated by piping the redacted stream into a small `risk_score.py`:
   - count of HIGH/CRITICAL findings in 30 days
   - certificate-failure rate
   - emergency-access-override frequency
   - top 5 unsigned binaries detected

**Deliverable.** A redacted, signed 30-day evidence bundle the buyer's CISO can take into the IC.

**Talk track.** "BGH 2021 ruling means raw patient data cannot move to the buyer pre-close. Observo solves this in the pipeline, not in the SIEM."

## Demo 2 (10 min) — Day-1 Dual-SIEM Fan-Out
*Maps to GTM motion #2, technical use cases 3.2 + 3.3.*

**Story.** "Deal closes Friday. Monday morning, both SOCs need full telemetry without breaking either side's detections."

**Steps.**
1. Show Observo pipeline with **two destinations active**:
   - **AI SIEM** (acquirer) via OTLP/HTTP → `https://xdr.us1.sentinelone.net/services/otlp/v1/logs`
   - **Microsoft Sentinel** (target) via Azure Monitor HTTP Data Collector
2. Apply the **OCSF normalisation processor** (only on the AI SIEM branch):
   - `event_category=patient_access` → `class_uid=6001`, `category_uid=6`
   - `severity=HIGH` → `severity_id=5`, `severity_str=HIGH`
3. Add an **enrichment processor** on both branches: `m_a_phase = "day1-dual"`, `legal_entity = "GroupB"`.
4. Trigger the pipeline. Live-tail a `MALWARE_DETECTED` event flowing.
5. Open both SIEMs side-by-side:
   - **AI SIEM:** open `/dashboards/bsi-nis2-healthcare-overview`. Show the panel updating.
   - **Sentinel:** open the same event in the existing Sentinel workbook (no rule changes).

**Deliverable.** Zero-rewrite Day-1 cutover: the target SOC keeps its existing Sentinel content; the acquirer SOC sees the same events in OCSF immediately.

**Talk track.** "The pipeline is the schema-translation layer. Neither SOC pays the rule-rewriting tax."

## Demo 3 (10 min) — NIS2 Evidence-Pack on Demand
*Maps to GTM motion #3, technical use case 3.5.*

**Story.** "Auditor calls Tuesday at 3pm. Show me the evidence pack for the ransomware event in the acquired clinic, mapped to NIS2 Art. 23 reporting fields."

**Steps.**
1. In SDL search, find an `INTRUSION_DETECTED` or `MALWARE_DETECTED` event from the dataset.
2. Trigger an Observo **on-demand pipeline** (or scheduled saved query):
   - input: SIEM webhook with the alert ID
   - look up correlated events ±2 h around the alert from cold-tier S3
   - join with asset inventory (entity tag → wE / bwE / KRITIS classification)
   - render a JSON evidence bundle following the BSI reporting template
   - hash + sign + drop into a "BSI submissions" WORM bucket
3. Open the bundle in a viewer. Highlight:
   - timestamps in UTC + Europe/Berlin
   - affected legal entity = **acquired clinic**
   - first-detected, last-seen, containment timestamps
   - SHA-256 of every source log file referenced

**Deliverable.** A reproducible, signed NIS2 evidence pack — produced in <60 seconds.

**Talk track.** "Article 23 NIS2: 24h early warning, 72h notification, 1 month report. The pack is the same artefact at every step — just more fields filled in."

## Demo 4 (10 min) — Cost-Out Workshop (post-Day-90)
*Maps to GTM motion #4, technical use cases 3.4 + 3.8.*

**Story.** "You're 90 days post-close. Combined SIEM ingest doubled. Procurement is asking for a cost plan."

**Steps.**
1. Show Observo's **volume analytics dashboard**:
   - ingest GB/day per source pre/post-merger (split by Avelios + Omniconnect)
   - top 20 noisiest event types
2. Apply three reduction processors live:
   - **DROP:** `event_type=KONNEKTOR_HEALTH_CHECK outcome=success` (drop ~25 % of TI volume)
   - **SAMPLE:** `event_type=USER_LOGIN_SUCCESS` 1:10 (drop ~40 % of auth volume)
   - **TIER:** route DEBUG/INFO that nobody alerts on directly to S3 Glacier (10-year WORM) instead of SIEM
3. Show the **SIEM-bound** byte counter dropping in real-time. Reference the public Observo case study (Sentinel: >50 % cost, >80 % storage savings).
4. Show that the dashboard `/dashboards/bsi-nis2-healthcare-overview` is **unchanged** — every panel still has data because the dropped events were noise.
5. Show a query against cold-tier S3 (Athena / DuckDB) to prove the dropped data is still recoverable for forensic / 10-year retention.

**Deliverable.** A 1-page "before / after" cost report; SIEM bill cut, NIS2 / B3S / gematik retention obligations preserved.

**Talk track.** "Retain everything for compliance. Index only what generates alerts. Pipelines decide."

## Demo 5 (10 min) — TI / Konnektor Compliance Routing
*Maps to GTM motion #5, technical use case 3.6.*

**Story.** "You inherited 200 Konnektors, 5 KIM gateways, 1 200 HBAs, 4 000 SMC-Bs across the merged group. gematik wants audit-grade logging; SOC wants security alerts; ops wants availability metrics. One pipeline, three destinations."

**Steps.**
1. Show Observo pipeline taking the **omniconnect** S3 prefix.
2. Apply a **classifier processor**:
   - `event_category=ti_connection` AND `severity in (HIGH,CRITICAL)` → AI SIEM (security)
   - `event_category=ti_connection` AND `outcome=success` → APM / Grafana (availability)
   - everything → **WORM bucket with Object Lock for 10 years** (gematik audit)
3. Show three live destinations updating from a single event stream.
4. Open `/dashboards/bsi-nis2-healthcare-overview` → **Omniconnect** tab. Highlight `CERTIFICATE_EXPIRED`, `KONNEKTOR_DISCONNECTED`, `VPN_TUNNEL_FAILED` panels.
5. Open the gematik audit bucket — show the WORM lock state.

**Deliverable.** A single pipeline serving SOC, NOC, and the gematik auditor — across the merged Konnektor fleet.

**Talk track.** "BSI TR-03116 + gematik specs require traceable, immutable logs for years. WORM Object Lock + Observo retention rules deliver that without paying SIEM rates for cold data."

## Demo 6 (10 min) — Joint Reference Architecture Sales Play
*Maps to GTM motion #6, technical use cases 3.7 + 3.9 + 3.10.*

**Story.** "Carve-outs, GDPR DSARs, supply-chain risk on inherited vendors — three things that hit hardest in Year 1 of a healthcare M&A. One pipeline pattern handles all three."

**Steps (3 mini-demos, ~3 min each).**

### 6a. Carve-out / TSA dual-tenant
- Show Observo with two RBAC-fenced tenant views.
- Same input bucket, classifier on `legal_entity` field.
- Tenant-A SOC sees only seller events; Tenant-B SOC sees only carved-out events.
- Toggle a "TSA expiry" rule that automatically stops Tenant-B routing on a calendar date.

### 6b. GDPR Art. 17 erasure accelerator
- Customer drops a KVNR into a search box.
- Pipeline scans hot SIEM index + cold S3 for any reference.
- Returns a DPO-grade report: "Found in Avelios HIS audit (3 events), Omniconnect KIM (1), Sentinel detection (0)."
- One click pseudonymises hot records and queues a deletion for cold tier.

### 6c. Supply-chain risk monitoring (NIS2 21(2)(d))
- Show a **vendor-tagged** stream: every event carries `vendor`, `product_version`, `contract_id`.
- Pipeline enriches with CVE feed + CISA KEV.
- Open a new dashboard tab "Supply Chain — Acquired Estate" listing inherited vendors, versions, and any KEV-listed CVEs against them.

**Deliverable.** A reference architecture diagram (Observo + AI SIEM + Sentinel + Splunk + S3 WORM) that the customer can hand to procurement and to BSI auditors.

**Talk track.** "M&A inherits not just systems but supplier risk. The pipeline is where you mark, monitor, and prove that risk is under control."
